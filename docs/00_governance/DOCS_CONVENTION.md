# Docs Convention

## Canonical Documents

- Architecture: `docs/10_architecture/ARCHITECTURE.md`
- SSoT: `docs/20_ssot/SSOT.md`
- Templates Index: `docs/25_templates/TEMPLATES_INDEX.md`
- Task Board: `docs/30_tasks/TASKS.md`
- Test Protocol: `docs/40_tests/TEST_PROTOCOL.md`
- Test Cases: `docs/40_tests/TEST_CASES.md`

## Naming Rules

- Canonical docs use fixed uppercase names (`ARCHITECTURE.md`, `SSOT.md`, etc.).
- Task files use: `TASK-YYYYMMDD-<slug>.md`
- Test run files use: `TEST_RUN-YYYYMMDD-HHMM.md`
- Do not use spaces in filenames.
- Do not encode versions in filenames; keep version in document body.

## Mandatory Pre-Read (for agents)

Before any command or file edit, read:

1. `docs/10_architecture/ARCHITECTURE.md`
2. `docs/20_ssot/SSOT.md`
3. `docs/25_templates/TEMPLATES_INDEX.md`
4. `docs/40_tests/TEST_PROTOCOL.md`
5. `docs/40_tests/TEST_CASES.md`
